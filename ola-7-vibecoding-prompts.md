## Why This Exists

Everyone chases the latest model. But if you don't know how to prompt, the best model won't save you. These 7 prompts cover the full vibe coding workflow — from structuring an idea before you write code, all the way to optimizing output before you ship. Ola's been using these for months and they work across any AI coding tool.

---

## How to Use These

1. **Pick the prompt** that matches what you're doing (the "Use when" label tells you)
2. **Replace `[paste]` and `[idea]`** with your actual code or idea
3. **Paste directly** into ChatGPT, Claude, Gemini, Cursor, Windsurf, or any coding LLM
4. **The more context you give**, the better the result
5. **Prompt 3 (Build Step-by-Step) is critical** for non-trivial projects — it prevents the AI from going off and building everything wrong at once

---

## 1. Structure Before Code

**Use when:** Starting a new project or feature

```
before writing any code, break this project into:

• core features
• user flow
• backend logic
• dependencies

don't write code yet. just structure it properly.
```

---

## 2. Think Like a Senior Dev

**Use when:** Evaluating an idea before building

```
act like a senior engineer reviewing this idea:

[idea]

what's wrong with it?
what will break?
what am i not thinking about?

be brutally honest.
```

---

## 3. Build Step-by-Step (CRITICAL)

**Use when:** Complex builds — prevents scope creep

```
don't generate everything at once.

we'll build this step by step.

start with the first component only.
wait for my confirmation before moving forward.
```

---

## 4. Debug Like a Pro

**Use when:** Something broke and you need to understand *why*

```
here's my code:

[paste]

don't just fix it.

explain:
• what's wrong
• why it broke
• how to avoid it next time
```

---

## 5. Improve Architecture

**Use when:** Codebase feels messy or fragile

```
review this codebase structure:

[paste]

suggest a cleaner, more scalable architecture.
remove unnecessary complexity.
```

---

## 6. Real-World Testing

**Use when:** Before shipping

```
assume this is deployed.

what are the 5 most likely ways this fails in real usage?
```

---

## 7. Optimize Output

**Use when:** Code works but could be better

```
here's the current version:

[paste]

make it:
• cleaner
• faster
• more maintainable

without changing the core logic.
```
